# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report them privately through GitHub's security advisories:

➡️ [Report a vulnerability](https://github.com/heronlabs/env-ssm/security/advisories/new)

This opens a private channel where we can discuss and fix the issue before any
public disclosure. If you are unable to use GitHub advisories, you may reach out
to the maintainers via the contact listed on the
[Heron Labs GitHub organization](https://github.com/heronlabs).

Please include enough detail to reproduce the issue (affected version, steps,
and impact). We aim to acknowledge reports promptly and will keep you updated on
the remediation.

## Supported Versions

Only the latest published version of `@heronlabs/env-ssm` on npm receives
security fixes. Please upgrade to the latest release before reporting an issue.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| older   | :x:                |
